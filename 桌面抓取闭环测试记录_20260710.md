# 桌面抓取闭环测试记录 2026-07-10

## 测试目标

本轮目标是把 HP60C RGB-D 视觉定位接入左臂桌面抓取流程，并让目标物随机摆放后，控制器能够根据 `arm_x/arm_y/arm_z` 自动调整肩前摆、肩侧摆、手肘和手腕姿态。

重点验证：

- HP60C RGB-D 单帧读取、ROI 扫描和 debug 图输出。
- 深度粗定位与 RGB 暗色目标二次修正结合。
- 放大桌面搜索区域，避免目标物出绿框后无法定位。
- 目标像素转换为机械臂粗坐标 `arm_x/arm_y/arm_z`。
- `arm_x` 接入控制器，使不同前后距离生成不同手腕、手肘和侧摆补偿。
- 使用 `--skip-claw` 先验证姿态，再完整执行压力闭合抓取。

## 推荐视觉命令

当前推荐使用放大后的默认 ROI，不再手动传旧的 `--roi-x0 220 --roi-y0 170 --roi-x1 440 --roi-y1 330`：

```bash
python3 hp60c_auto_target.py \
  --scan-windows \
  --window-w 80 \
  --window-h 85 \
  --window-step 20 \
  --min-area 30 \
  --max-area 6000 \
  --aim-y-frac 0.25 \
  --target-depth-percentile 75 \
  --save-debug-prefix test_auto
```

debug 图标记：

- 绿色框：当前搜索 ROI。
- 黄色圈：最终目标瞄准像素。
- 青色圈：深度粗定位像素。
- 紫色框：RGB 暗色目标修正区域。

## 视觉到机械臂坐标

当前 HP60C 自动定位脚本输出：

```text
arm_x = camera_x + 18.4
arm_y = camera_y + 20.9
arm_z = camera_z - 27.4
```

RGB 暗色修正启用后，会重新在最终像素附近取深度，并输出：

```text
depth_before_rgb_refine_m
target_depth_source = "rgb_refined_pixel_patch"
```

## 控制器关键更新

`left_arm_controller.py` 本轮新增或稳定了以下行为：

- `target --skip-claw`：执行到手腕姿态后停止，不闭合爪子。
- `target --front-bias-deg/--side-bias-deg/--elbow-bias-deg/--wrist-bias-deg`：临时人工偏置。
- Home 动作先打开爪子，再回手腕、侧摆、手肘和前摆，降低回家时碰撞风险。
- `arm_x` 控制粗前后距离，不再只由 `arm_z` 和 `arm_y` 决定姿态。
- 侧摆、手肘、手腕补偿根据 `arm_x` 自适应。

当前主要经验参数：

```text
ARM_X_BASELINE_CM = 8.8
ARM_X_ADAPT_NEAR_CM = 8.0
ARM_X_ADAPT_FAR_CM = 12.0

ARM_Y_CM_PER_DEG = 1.4
ARM_Y_NEAR_INWARD_BIAS_DEG = -2.0
ARM_Y_FAR_INWARD_BIAS_DEG = -10.0

WRIST_NEAR_EXTRA_DOWN_BIAS_DEG = 12.0
WRIST_FAR_EXTRA_DOWN_BIAS_DEG = 22.0
WRIST_FAR_X_UP_START_CM = 14.0
WRIST_FAR_X_UP_FULL_CM = 18.0
WRIST_FAR_X_UP_MAX_DEG = 10.0

ELBOW_NEAR_EXTRA_DOWN_BIAS_DEG = 0.0
ELBOW_FAR_EXTRA_DOWN_BIAS_DEG = 9.0
```

## 成功抓取样本

### 样本 1：低位近端

```text
target --x 6.47 --y 29.54 --z 28.20
arm_x_adapt_t = 0.0
arm_y_inward_bias_deg = -2.0
elbow_extra_down_bias_deg = 0.0
wrist_down_offset_deg = -95.18
```

结果：

```text
claw contact detected pos=4.5653 tau=0.3883
safe path stopped after eight motions. Object should be lifted.
```

状态：完美抓取。

### 样本 2：低位偏外

```text
target --x 13.84 --y 26.46 --z 31.50
arm_x_adapt_t = 1.0
arm_y_inward_bias_deg = -10.0
elbow_extra_down_bias_deg = 9.0
wrist_down_offset_deg = -79.30
```

结果：

```text
claw contact detected pos=5.3264 tau=0.3394
safe path stopped after eight motions. Object should be lifted.
```

状态：完美抓取。

### 样本 3：远端低位

```text
target --x 17.20 --y 24.16 --z 33.20
wrist_far_x_up_deg = 8.0
wrist_down_offset_deg = -68.24
```

结果：

```text
claw contact detected pos=6.0637 tau=0.3004
safe path stopped after eight motions. Object should be lifted.
```

状态：完美抓取。

### 样本 4：中近端低位

```text
target --x 8.37 --y 24.78 --z 33.20
arm_x_adapt_t = 0.0925
arm_y_inward_bias_deg = -2.74
elbow_extra_down_bias_deg = 0.8325
wrist_down_offset_deg ~= -76.1 after wrist adaptive update
```

测试反馈：手腕需要上抬，因此将手腕额外下压从固定 `22 deg` 改为近端 `12 deg`、远端 `22 deg` 的插值。

## 当前结论

截至本轮结束，闭环已经可以在多个随机位置完成：

- 自动检测目标物。
- 生成目标机械臂坐标。
- 根据 `arm_x` 自动切换近端、中端、远端补偿。
- 压力闭合检测目标接触。
- 抓取后抬起目标物。

低位桌面抓取已经从单点调参进入多点经验曲线阶段。下一轮重点应继续测试 `x=8-12 cm` 的中间过渡区，并确认近端手腕下压从 `22 deg` 改为自适应后是否稳定。

## 下一步

- 连续随机摆放测试 5-10 次，记录失败样本。
- 若中间区稳定，把 `hp60c_auto_target.py --execute` 作为默认一键流程。
- 若出现误检，优先检查 debug 图中黄色圈、紫色框和青色圈的相对位置。
- 之后再考虑加入更完整的几何 IK，而不是继续只靠经验插值。
